DTSCA: Dynamic Threshold & Saliency-Driven Cognitive Architecture
(动态阈值与显著性驱动的认知架构)

💡 核心动机 (Motivation)
当前的 LLM（大语言模型）虽然展现出了强大的自然语言处理能力，但其底层依然依赖于“基于静态概率的高级数学推算”。它们缺乏人类认知机制中最核心的环节：动态注意力过滤与意识觉醒机制。

本项目（DTSCA）旨在提出一种全新的 AGI 逻辑架构。我们从神经科学中的“感觉门控（Sensory Gating）”与“显著性网络（Salience Network）”中汲取灵感，探讨如何通过一套动态阈值系统，让 AI 摆脱持续的线性计算，实现类似人类从“无意识后台运行（System 1）”到“显意识深度思考（System 2）”的动态跃迁。

🧠 核心逻辑架构 (Core Architecture)
本架构将 AI 的信息处理机制划分为四个动态流转的模块：

感知层 (Perception Layer)： 接收所有内外部的“因素（Factors）”（对应人类的内外感知）。

动态阈值监控器 (Dynamic Threshold Gating)： 类似大脑的前额叶皮层与网状激活系统。该阈值并非固定值，而是根据当前任务的“生存警觉（异常状态）”或“选择性专注（高负载任务）”实时浮动。

显著性触发器 (Saliency Trigger / "火星子")： 当某个因素的信息量超越了当前设定的动态阈值时，系统触发警报（即意识中的“火星子”），中断底层常规的概率输出。

双系统处理引擎 (Dual-System Engine)：

System 1 (后台自动程序)： 快速、低能耗。处理未打破阈值的常规输入（即非情绪性行为/本能反应）。

System 2 (深度推理模块)： 昂贵、高耗能。当被显著性触发器激活后，强行调配庞大算力进入逻辑推演与反思模式（即情绪性行为/显意识思考）。

习惯化下放机制 (Habituation & Sedimentation)： System 2 经过深度推理得到的结论，在验证稳固后，会被选择性地“降维”固化到 System 1 中，形成新的自动化处理逻辑，完成认知闭环。

🚧 工程挑战与攻坚方向 (Technical Mapping & Challenges)
从认知哲学走向底层代码，我们目前正在攻克以下三大核心壁垒。欢迎全球开发者共同参与讨论与共建：

难点一：显著性触发的“数学常量化”

挑战： 机器没有主观的“感受质（Qualia）”。如何定义一个输入的“重要性”或“异常值”？

探索方向： 尝试引入“信息熵突增（Entropy Surge）”或“模型预测置信度断崖式下降（Confidence Drop）”作为触发 System 2 的数学标尺。

难点二：克服习惯化过程中的“灾难性遗忘” (Catastrophic Forgetting)

挑战： 将 System 2 的深度思考经验实时固化为 System 1 的底层权重时，极易覆盖原有知识。

探索方向： 设计双层记忆架构（Dual-Memory Architecture），引入极其温和的“记忆蒸馏”机制，让新逻辑在沙盒中沉淀后再并入主干。

难点三：从哲学理念到算法实现的翻译鸿沟

挑战： 顶层架构设计极其复杂，底层逻辑容易演变为不可读的黑盒。

探索方向： 建立严格的人机协作规范与代码审查机制，确保架构在推进时保持高度透明。

🤝 贡献与协作规范 (Contribution Guidelines)
本架构高度关注逻辑的可读性与工程的透明度。所有的 PR (Pull Request) 与代码提交必须遵循以下协议：

拒绝黑盒代码： 所有代码必须附带极其详尽的逐行逻辑解释，清晰说明该段代码在 DTSCA 架构中对应的具体模块（如：这串代码是在模拟显著性网络还是在执行记忆蒸馏？）。

潜在风险前置： 提交的功能模块必须显式列出所有潜在的报错源泉（包括但不限于网络环境依赖、特定软件版本冲突、代码本身的边界错误等），以确保架构层面的绝对掌控权。
